# ساختار ساده دیتابیس انیمه

فعلاً فقط **یک جدول** داریم به اسم `anime`. هیچ جدول دیگری و هیچ رابطه‌ای لازم نیست.

## جدول `anime`

| ستون         | نوع        | توضیح                    |
|-------------|------------|---------------------------|
| `id`        | uuid       | شناسه یکتا (خود Supabase می‌سازه) |
| `title`     | text       | عنوان انیمه              |
| `cover_image` | text     | لینک تصویر جلد           |
| `synopsis`  | text       | خلاصه داستان (اختیاری)  |
| `format`    | text       | وضعیت/نوع: مثلاً TV، MOVIE (اختیاری) |
| `average_score` | number | امتیاز مثلاً ۸.۵ (اختیاری) |
| `episodes`  | integer    | تعداد قسمت‌ها (اختیاری)  |
| `studio`    | text       | استودیو (اختیاری)        |
| `season`    | text       | فصل پخش، مثلاً زمستان ۱۴۰۳ (اختیاری) |
| `start_date`| text       | تاریخ شروع پخش (اختیاری) |
| `end_date`  | text       | تاریخ پایان پخش (اختیاری) |
| `created_at`| timestamp  | زمان ثبت (خودکار)         |

## چطور اعمال کنم؟

**جدول جدید:** فایل `supabase-simple-schema.sql` را در SQL Editor اجرا کن.

**جدول از قبل داری و فقط می‌خواهی فیلدهای جدید اضافه شوند:** فایل `supabase-add-anime-fields.sql` را در SQL Editor اجرا کن (فیلدهای episodes، studio، season، start_date، end_date اضافه می‌شوند).

## ارتباط با اپ

- اپ فقط از همین جدول `anime` خوندن می‌کند.
- هیچ جدول جدا برای ژانر، استودیو، قسمت و… لازم نیست؛ همه‌چیز در همین یک جدول است.

بعداً اگر خواستی ژانر یا قسمت اضافه کنی، می‌توانیم مرحله‌به‌مرحله جدول/ستون جدید اضافه کنیم.
